# 练迹 · 本地健身记录

一个手机优先、本地优先的 React + TypeScript PWA。应用不需要账号、业务服务器或数据库；训练、身体和恢复数据默认只保存在当前浏览器中。仓库不含真实用户资料、历史记录、API Key、私有部署配置或旧 Git 历史。

## 已实现

- 按星期自动显示上肢 A、下肢 A、上肢 B、下肢 B 或休息日
- 93 个家庭训练动作数据库，含做法、提示、常见错误、肌群、部位、效果和安全提醒
- 自定义每日训练支持多关键词检索、分类筛选、收藏、最近使用、多选加入、替换和重排
- 每组实际次数、完成状态、RIR、跳过原因和训练备注
- 完成一组后自动启动组间休息计时器，可暂停、加 30 秒或提前结束
- 训练中按动作参数自动倒计时，并在最后 3 秒与结束时播放不同提示音
- 睡眠不足 6 小时自动减组；连续两晚不足 6 小时建议取消正式力量训练
- 历史训练、完成率和引体向上/俯卧撑/深蹲趋势
- 体重、围度、周平均值和每 4 周围度提醒
- 睡眠、疲劳、酸痛和关节疼痛恢复打卡
- 修改个人资料、器械和每个动作的组数、次数、重量、RIR、休息时间与个人备注
- 浅色、深色和跟随系统模式
- JSON 完整备份导入/导出、二次确认清空
- 可选 AI 教练：API Key 与对话只保留在当前浏览器会话，默认只发送计划和匿名统计汇总；详细健康上下文必须由用户主动开启
- Web App Manifest、离线缓存和手机桌面安装

## 安装依赖

需要 Node.js 20.19+ 或 22.13+。

```bash
pnpm install
```

没有 pnpm 时也可以使用：

```bash
npm install
```

## 电脑预览

开发模式：

```bash
pnpm dev
```

浏览器打开终端显示的 `http://localhost:5173`。开发服务器已监听局域网地址。

检查包含离线功能的生产版本：

```bash
pnpm build
pnpm preview
```

浏览器打开 `http://localhost:4173`。Service Worker 只在生产构建中注册，避免开发期间缓存旧代码。

## 手机打开

1. 确保手机和电脑连接同一个 Wi-Fi。
2. 执行 `pnpm dev`。
3. 在电脑终端找到 `Network` 后的地址，例如 `http://192.168.1.8:5173`。
4. 用手机浏览器打开该地址即可预览和打卡。

局域网的普通 HTTP 地址可以预览，但手机浏览器通常不允许它注册 Service Worker，因此不能可靠地离线安装。要安装正式 PWA，请把 `pnpm build` 生成的 `dist` 文件夹发布到任意 HTTPS 静态托管服务；这只是静态文件托管，不是后端，应用数据仍只留在手机本机。

## 安装到手机桌面

- Android Chrome/Edge：打开 HTTPS 地址，点页面“设置 → 安装”，或浏览器菜单中的“安装应用/添加到主屏幕”。
- iPhone/iPad Safari：打开 HTTPS 地址，点“分享” → “添加到主屏幕”。

安装后至少联网打开一次，所需文件会缓存到设备。之后可以离线打开。

## 测试

```bash
pnpm test
```

测试覆盖训练计划读取、关键参数、训练记录本地保存以及 JSON 备份往返导入。

Windows 上已安装 Edge 或 Chrome 时，还可以先运行 `pnpm preview`，再在另一个终端执行：

```bash
pnpm smoke
```

该测试会用 390 × 844 的手机视口点测五个页面、组间计时器，并在同一浏览器会话中模拟断网刷新。

## 数据安全

- 浏览器清除站点数据会删除本机记录。
- 换手机、换浏览器或清理数据前，在“设置 → 备份与数据”中导出 JSON。
- 备份包含训练、身体、恢复和个人资料，只应保存在可信设备中。
- 导入只接受不超过 5 MB 的 JSON 文件，会整体替换当前数据，并在执行前要求确认。
- 默认资料不预填生日、身高、体重、围度、蛋白目标或器械；首次体重记录会在本机建立起点。
- AI 教练请求固定发送至 DashScope 官方端点，禁止重定向、Cookie、缓存和 Referrer；姓名不会发送。
- 最少数据模式不发送年龄、身体指标、恢复记录、自由文本备注或逐次训练历史。详细模式会在界面中明确说明发送范围。
- 项目没有分析 SDK、广告 SDK、账号登录、远程日志或服务端数据同步。

更多边界与报告见 [PRIVACY.md](PRIVACY.md)、[SECURITY.md](SECURITY.md) 和 [SECURITY_REVIEW.md](SECURITY_REVIEW.md)。

## 公开发布

提交到公开仓库前建议执行：

```bash
pnpm install --frozen-lockfile
pnpm test
pnpm build
pnpm audit --prod --registry https://registry.npmjs.org
```

不要提交 `.env*`、证书、凭据文件、`dist/`、`node_modules/`、本地托管平台状态或任何用户导出的备份。仓库已提供 GitHub Actions，在每次 push 和 pull request 时执行测试与构建。
