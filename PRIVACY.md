# Privacy

练迹采用本地优先设计。项目本身不提供账号、数据库、分析 SDK、广告 SDK、远程日志或跨设备同步。

## 默认保存在浏览器中的数据

以下内容会保存在当前站点的浏览器存储中：

- 训练计划、训练草稿与历史记录
- 睡眠、疲劳、酸痛和疼痛记录
- 体重、围度与自由文本备注
- 用户自行填写的称呼、生日、身高、目标和器械
- 主题、收藏、最近使用动作与教程设置

清除浏览器站点数据会删除这些内容。JSON 备份包含上述数据，导出前会明确提示，只应保存在可信设备中。

## AI 教练

AI 教练是可选功能。未填写 API Key 时不会发起 AI 请求。

- API Key 只保存在 `sessionStorage`，关闭浏览器会话后通常会失效。
- 对话只保存在当前 `sessionStorage` 会话。
- 请求固定发送到 `https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions`。
- 姓名不会加入任何 AI 请求。
- 默认“最少数据”模式只发送器械、当前训练计划和匿名统计汇总，不发送年龄、身体指标、恢复记录、自由文本备注或逐次训练历史。
- “详细数据”模式必须由用户主动开启，会发送界面中列出的健康与训练上下文。
- 用户直接输入到提问框的文字会原样发送，因此不应输入不希望交给服务商处理的信息。

第三方 AI 服务对请求的处理受其自身条款与隐私政策约束。本项目无法代替服务商承诺数据保留方式。

## 外部内容

动作教程可能在用户操作后打开或嵌入哔哩哔哩播放器。访问第三方内容时，第三方可能接收常规网络信息。站点设置了 `Referrer-Policy: no-referrer`，并通过 CSP 限制允许连接与嵌入的域名。

## 公开仓库边界

公开仓库不应包含真实用户备份、`.env`、证书、密钥、托管平台本地状态、构建缓存、旧版本压缩包或包含个人数据的截图。相关模式已加入 `.gitignore`，但提交前仍应执行人工检查和密钥扫描。
